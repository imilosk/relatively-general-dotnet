Free Anti Virus on Google

by Oren Eini

posted on: June 13, 2005

This is the first result when searching for "Free Anti Virus" on Google:  AVG Anti Virus: HOME The complex anti-virus protection for your computer. I've heard about AVG (but couldn't recall the name), but the description is quite worrying. I hope that this is a typo and they meant complete. Their home page says: "Tough on viruses, easy on users." So I think that this is indeed a typo. They have a free version here, and the restrictions on install seems reasonable (personal, non commercial use only) and the restrictions on use don't bother me at all (one scheduled scan per day, one update per day, no advance tuning, can't install on server, can't scan networked drives, etc). Unfrotantely, I can't install it because it barfed on install: Installer initialization failed due to following error:     Undefined error: Access is denied.  (5) I guess I will have to reboot anyway :-( if only to remove Norton Anti Virus from my computer, it now refuse to uninstall if I won't rebbot.

Microsoft Acrylic: The Five Minutes Review

by Oren Eini

posted on: June 13, 2005

I downloaded the new beta of MS Acrylic, (70Mb), installed and opened it. The first thing that I noticed is that the background canavas for the application is filled with "BETA"s, just to make sure that we would get that this is beta software, apperantly, if we missed it in the site and installer. Then the reall annoyance started. Acrylic uses the oh so useless notion of floating panels, I has strong dislike to those, even though most graphical programs seems to find them essensial. A properly docked panels are a different story, as evidenced by VS.Net. The real problem was that the Acrylic started at one monitor, and all the panels started on the second one. What is more, you can't get them to dock. I took a very cursory look, and it seems to me that this is an application that is targeted at the Photoshop users. I have no need of yet another image editing, I can use Paint & GIMP, and that is mostly enough for me. (And for others as well :-) ) I hoped that it would have SVG capabilities, as the only editor that I've for that is Inkscape, which come with a whole set of strange UI design issues.

Scripting Applications: Not Just Smalltalk

by Oren Eini

posted on: June 12, 2005

James Robertson has a post about application scripting that I partly agree with.  That's why I include a workspace in BottomFeeder - I actually use it myself for scripting and experimenting. That kind of end application scripting has led to actual product features, and is something that is well nigh impossible in Java or C#. In applications shipped in those languages, you have a dead end - if the developer didn't consider it, you're just done. Does this mean that just anyone can start coding in the Bf workspace? No, you do need to know the APIs and class names - that's a limitation. But at least you get far enought to have the limitation :) I agree that application scripting at the client is useful; and I think that it's very likely that users / developers would want an easy way to modify the application after it has shipped. My problem is that he goes on to claims that it's nigh impossible to do so in C# or Java. No I can't say much for Java but in .Net you can most certainly do that, and it's not that much of a bother. Here is my post about Automating Your Application and here is the original article about adding scripting capabilities to your application.
